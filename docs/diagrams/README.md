# Diagrams Folder

This folder contains all diagrams for Lab 2 Part 3.

## Required Diagrams

1. **Architecture Model** (`architecture_model.png/.puml`)
   - 3-tier architecture
   - Component interactions
   - Technology stack per layer

2. **Entity Relationship Diagram** (`erd.png/.puml`)
   - All entities with attributes
   - Primary keys and foreign keys
   - Cardinality relationships

3. **Use Case Diagram** (`use_case_diagram.png/.puml`)
   - Actors: User, Authenticated User
   - All use cases
   - Include/extend relationships

4. **Class Diagram** (`class_diagram.png/.puml`)
   - Model classes
   - Service classes
   - Controller classes
   - Attributes and methods
   - Relationships

5. **Sequence Diagrams**
   - `sequence_load_edit_cv.png/.puml` - Load & Edit CV flow
   - `sequence_export_cv.png/.puml` - Export CV flow
   - `sequence_login.png/.puml` - Login flow

## Tools

Recommended tools (free and open-source):
- **Draw.io (diagrams.net)**: Web-based, easy to use
- **PlantUML**: Text-based diagrams, good for version control
- **Lucidchart**: Free tier available
- **StarUML**: Desktop application

## Diagram Guidelines

- Use consistent styling across all diagrams
- Include legends where necessary
- Use clear, readable fonts (minimum 10pt)
- Export in high resolution (PNG or SVG)
- Keep source files (.drawio, .puml) for future edits
- Follow UML standard notation
